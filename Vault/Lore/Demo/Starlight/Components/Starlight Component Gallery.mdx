:::

## Typography system

The codex uses **Cinzel** as the display face for the main title layer, **Source Serif 4** for long-form prose, **IBM Plex Sans** for interface and metadata, and **IBM Plex Mono** for code or technical fragments.

[cols:2-1 gap=lg]
[col]
# Cinzel H1 Display

## Source Serif H2 Archive Layer

This body copy should feel like a public codex: readable, literary, and archival without becoming too ornate. It needs to carry CITADEL prose and ENTROPY technical notes without forcing either era to dress as the other.

### IBM Plex Sans lower heading

Small labels, navigation, metadata, tables, captions, and structural UI should feel cleaner and more industrial than the prose.
[/col]

[col]
[card:accent]
### Font roles

- Display: `Cinzel`
- Body: `Source Serif 4`
- UI: `IBM Plex Sans`
- Code: `IBM Plex Mono`
[/card]
[/col]
[/cols]

## Codex formatting tags

The tags below are transformed during the Obsidian-to-Starlight sync step. They are written like BBCode, but rendered as responsive layout HTML with controlled CSS classes.

## Mathematical notation

Inline TeX can be written like GitHub-style Markdown: $R(t)=R_0e^{-\lambda t}$.

Display TeX can be written with double-dollar delimiters:

$$
R(t)=R_0e^{-\lambda t}
$$

Fenced `math` blocks are converted into display equations at runtime:

```math
\begin{aligned}
\mathcal{R}_{total}
  &= \sum_{i=1}^{n} \alpha_i \psi_i(t) \\
  &= \alpha_1 \psi_1(t) + \alpha_2 \psi_2(t) + \cdots + \alpha_n \psi_n(t)
\end{aligned}
```

Use the codex equation wrapper when a formula needs a title or article-level panel treatment:

````md
[equation:title="Resonance decay model"]
```math
R(t)=R_0e^{-\lambda t}
```
[/equation]
````

```math

```
[/equation]

## Steps

<Steps>

1. Write lore in `Vault/Lore` using normal Markdown.

2. Use codex tags such as `[cols:2-1]`, `[row]`, `[col:12 md:8]`, `[card]`, `[note]`, `[warning]`, `[lore]`, and `[equation]` when you need layout.

3. Run the sync and build commands.

   ```bash title="Build test"
   cd Site
   npm run sync
   npm run build
   ```

4. Check the public site and tune CSS only once, not per article.

</Steps>
