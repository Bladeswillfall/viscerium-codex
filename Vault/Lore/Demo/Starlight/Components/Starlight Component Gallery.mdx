## Calendar components

The full Okse calendar is now placed with a shortcode, not automatically injected into the page title area. This allows the calendar to appear wherever the note author places it.

```md
[Calendar:ID-0001]
```

[Calendar:ID-0001]

The page also declares an intercalary leap-day `calendarDate`, so the automatic date badge should show `Engimanutur 02` for year `20004`.

## Typography system

The codex uses **Cinzel** as the display face for the main title layer, **Source Serif 4** for long-form prose, **IBM Plex Sans** for interface and metadata, and **IBM Plex Mono** for code or technical fragments.

[cols:2-1 gap=lg]
[col]
# Cinzel H1 Display

## Source Serif H2 Archive Layer

This body copy should feel like a public codex: readable, literary, and archival without becoming too ornate. It needs to carry CITADEL prose and ENTROPY technical notes without forcing either era to dress as the other.

### IBM Plex Sans lower heading

Small labels, navigation, metadata, tables, captions, and structural UI should feel cleaner and more industrial than the prose.
[/col]

[col]
[card:accent]
### Font roles

- Display: `Cinzel`
- Body: `Source Serif 4`
- UI: `IBM Plex Sans`
- Code: `IBM Plex Mono`
[/card]
[/col]
[/cols]

## Mathematical notation

Inline TeX can be written like GitHub-style Markdown: $a^2+b^2=c^2$.

Display TeX can be written with double-dollar delimiters:

$$
a^2+b^2=c^2
$$

[equation:title="Resonance decay model"]
```math
R(t)=R_0e^{-\lambda t}
```
[/equation]
