# Security policy

## Supported version

The production deployment from the `main` branch is the supported version of the VISCERIUM Codex.

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability that could expose visitors, deployment credentials, unpublished material, or repository access.

Use GitHub's private vulnerability reporting feature when it is available for this repository. If that feature is unavailable, contact the repository owner privately through the contact route published on the VISCERIUM Codex.

Include:

- the affected URL, file, workflow, or dependency;
- clear reproduction steps;
- the likely impact;
- any proof-of-concept material needed to confirm the issue;
- whether the issue is already public.

Do not access data that is not yours, disrupt the production site, run destructive tests, or attempt to obtain deployment secrets. Non-destructive verification is welcome.

The maintainer will acknowledge a credible report as soon as practical, investigate it, and coordinate remediation before public disclosure where possible.
